# Data Folder

This folder is where input and output data lives. **No data files are committed to this repository** — all data is fetched from the original sources or generated by the pipeline. Follow the steps below to populate it before running the pipeline.

## Structure

```
data/
├── raw/         # input files (gitignored). You populate this.
├── processed/   # pipeline output (gitignored, regenerated on every run).
└── sample/      # placeholder for any small sample files (currently empty).
```

## How to populate `raw/`

The pipeline expects six files in `data/raw/`:

| File | Source | Size | How to obtain |
|---|---|---|---|
| `games.csv` | Kaggle | ~5 MB | Download from <https://www.kaggle.com/datasets/antonkozyriev/game-recommendations-on-steam> and copy to `data/raw/`. |
| `games_metadata.json` | Kaggle | ~18 MB | (same Kaggle dataset) |
| `recommendations.csv` | Kaggle | ~2 GB | (same Kaggle dataset) |
| `steamspy_data.csv` | SteamSpy public API | ~10 KB | Run `python -m src.ingestion.download_steamspy` from the project root. |
| `external_genres.csv` | curated lookup | <1 KB | Create manually, two columns (`genre,category`). See `docs/data_dictionary.md` § 1.5 for the expected format. |
| `wikidata_games.json` | Wikidata SPARQL | ~2 MB | Run `python -m src.ingestion.download_wikidata` from the project root. |

## Why nothing is committed here

* **License**: the Kaggle Steam dataset has its own license and cannot be redistributed through this repository.
* **Size**: `recommendations.csv` is ~2 GB, well above GitHub's 100 MB single-file limit.
* **Reproducibility**: SteamSpy and Wikidata are fetched live, so committing a stale snapshot would mislead future readers.

After all six files are in place under `data/raw/`, run `python -m src.main` from the project root. Output appears in `data/processed/result_parquet/` and a Markdown data-quality report is written to `docs/validation.md`.
